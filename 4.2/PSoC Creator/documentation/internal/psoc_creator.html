Unlike fixed function microcontrollers, PSoC allows you to choose the on-chip peripherals you need. Instead of settling for a part that has most of what you need and a lot of what you don't, with PSoC you simply configure the device with your chosen functionality. No more, no less.

And the configuration process couldn't be easier. You choose peripherals from a catalog of pre-defined and pre-tested components and simply drag them into your design schematic. The components range from simple gates for glue logic, through complete implementations of digital and analog peripheral functions, up to sophisticated communications and applications solutions like USB and CapSense. Then choose your clock frequencies, connect peripheral I/O to pins, and you're done. The built-in configuration tools map your design into the device and calculate the clock setup and routing automatically.

But this is not the complex hardware configuration process you've seen before. All components are parameterized so that the details of the implementation are abstracted away. You set up components in terms that are natural to their function; such as the baud rate for a UART, power and gain for an amplifier, or duty cycle of a PWM.

You are about to see that, like no other platform available today, PSoC Creator and PSoC form a uniquely flexible combination to design and build your own system-on-chip in less time than it takes you to read the competitors' data sheets!

An embedded device is only as good as the software controlling it. With those old-fashioned fixed function microcontrollers, that software can be really difficult to get right. There is assembler code that won't port to new architectures, memory maps that vary between devices, and peripherals with complex and confusing register sets. PSoC Creator avoids those problems by combining a C-based development flow with automatically generated software APIs for the components in your design.

A C-based flow enables portability across both the PSoC3 and PSoC5 CPU architectures. We guarantee that our boot code and component APIs can be re-targeted to new devices, regardless of the CPU, with just a re-build of the application. We even provide free compilers with no restrictions on use, application size or the time you can use it.

Our APIs make controlling your on-chip peripherals simple and fast. They are consistently named so you soon get used to using them and can often work with a new component without even opening the datasheet to check the software interface. You don't need to learn esoteric register names and write ugly macros to manipulate bit-fields. The APIs reduce coding errors and ensure correct interaction with the peripheral so the whole software development is faster, easier and less error prone.

Helping you write this code is a built-in modern software editor - providing popular features like keyword coloring, brace-matching, outlining, auto-commenting, auto-indenting, and code snippets - that combines with a powerful Workspace Editor to enable the rapid development of sophisticated applications for PSoC devices.

When it comes time to debug the application, PSoC Creator has all the modern features of a cross-debugger plus a unique peripheral debug window that shows you the status of the internals of your on-chip components. Other useful features include the "hover" function that shows you the type, address and value of any variable you hover over with the mouse and the ability to set (and remember) breakpoints in your code before even starting the debugger.

Application control is provided by the usual set of execute and stepping functions and a comprehensive suite of windows adds unbeatable visibility into the device's state. In addition to the C and disassembly windows, there are registers and memory, call stack and locals, plus memory and watch windows. All windows can be moved around to create the perfect setup for you and data can be viewed in any desired format and edited in-place.

Host-to-device connectivity is provided by our MiniProg3 programmer/debugger, which connects the PC's USB port to the device JTAG interface. A MiniProg3 is included in every evaluation, development or solution-focused kit we provide and more can be purchased separately. MiniProg3 and every PSoC3 and PSoC5 family support 5-pin JTAG plus the less intrusive 2-pin serial wire option. A compatible set of on-chip instruction and data breakpoint resources ensure a consistent debugging experience regardless of the target CPU architecture.

Central to the principles behind PSoC Creator is design reuse. The encapsulation of a working design into a reusable component makes it quick and easy to create future designs with fewer errors and less cluttered schematics.

All IDEs may support software libraries - be they from the compiler vendor, a third-party, or developed in-house - but a PSoC Creator library can also include hardware configurations. All those board layouts you've made in the past and have carefully integrated into a PSoC device to save money, footprint and time can now be safely stored in a library of your own components to reuse as often as you like in your future designs.

Creating your own components is as easy as drawing the design and having the tool generate a symbol to represent it. Place this component into a library and you're ready for reuse. Any design project that references the library will see the new component in the catalog and use it just like you would any other.

The components you create are full peers of the components we supply with the tool. You can extend them beyond simple schematics by adding your own software API code or using a parameter to give the end-user control over the component setup without opening up the implementation. By leveraging design reuse within PSoC Creator you can quickly build your libraries of designs so that, with each new project you take on, the focus is always on the innovation, not the legacy functionality.
